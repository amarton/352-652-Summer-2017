Intro to Javascript/JQuery Libraries
Today's exercise will serve as your introduction to using the thousands of Javascript and JQuery libraries written by more experienced coders.
Libraries are packages of code that make it easy to do complicated things without having to write a bunch of custom code yourself.
There are libraries that do pretty much anything you can think of:
  • Build basic charts and graphs (Highcharts)
  • Build INSANE charts and graphs (D3)
  • Pop up maps and text and other information on a page in time with news video! (Popcorn.js)
  • Full screen background video (Bigvideo.js)
  • My favorite: Bacon Text Wrap (Bacon.js)
You can find here and here and here. Or by googling "jquery library" and the thing you want it to do. Many developers host their libraries on Github.
Today's Library: FooTables
Today we'll be using a library called FooTables, which allows you to easily create sortable data tables that hide columns when the browser shrinks to make it display better on mobile.
We'll be building this sortable table of all the high schools in the country that have the name Redskins as their mascot. It was part of this project by Capital News Service project, The Other Redskins.
While the step-by-step instructions here apply specifically to using the FooTable library, you should expect to follow similar steps to make any other javascript/jquery library work for you
Step 1: Downloading the Library
Every library will have a package of source code files that make the whole thing work. At a minimum, the package will contain files with this extension:
  • .js (javascript, sometimes one file, sometimes multiple files, often contained in a folder called js or javascript)
Sometimes they will contain:
  • .css (specific styles)
  • .html (usually a page of examples of ways to use the library
There are often other types of files in there too, but for now, just worry about js, css and html.
Imagine we're building a car. These files are the engine of the car.
So, anytime you want to use a library, the first step is to download it. Download the Foo Table library now and unzip it.
Step 2: Building our Table
In the folder, save a blank stylesheet in the css folder (style.css - and DO NOT USE CSS RESET TODAY) and a blank html document in the root folder (index.html).
Just like normal, build a valid HTML document and include the stylesheet. Sorry about the indenting. This code editor isn't working on me.
<!DOCTYPE HTML>

<head>
<title>Redskins Table Data</title>
<link rel="stylesheet" type="text/css" href="css/style.css" />
</head>

<body>
</body>

</html>

Step 3: Hook up the Library
In the car analogy, the library is the engine and the HTML document and table is the frame of the car. The first step in getting our car to work is to bolt the engine under the hood.
We do this by including the library's core javascript files and css files in the head tag like so:
<!DOCTYPE HTML>

<head>
<title>Redskins Table Data</title>
<link rel="stylesheet" type="text/css" href="css/style.css" />
<link href="css/footable-0.1.css" rel="stylesheet" type="text/css" />
<script src="js/footable-0.1.js" type="text/javascript"></script>
 </head>

You're used to including CSS files at this point. Including javascript works the same way, but instead of <link> we use <script>
Open up Footable-0.1.js and take a look at it. Pretty scary, huh! For now, you don't need to know how to read any of that. Just know that core features of this library are contained within.
Take a look at Footable-0.1.css. It's just some special styles that apply to this library. It may look a little more complicated then you're used to. But it's just CSS. Remember, not every library will have CSS files.
Many libraries will store all of the source code in one javascript file. Some will break it into multiple files. FooTable is one of these. The core features are contained in the files we've just included. But the sorting functionality is contained in a separate javascript file, and is styled by a separate CSS file.
Let's include both of these now:
<!DOCTYPE HTML>

<head>
<title>Redskins Table Data</title>
<link rel="stylesheet" type="text/css" href="css/style.css" />
<link href="css/footable-0.1.css" rel="stylesheet" type="text/css" />
<link href="css/footable.sortable-0.1.css" rel="stylesheet" type="text/css" />
<script src="js/footable.sortable.js" type="text/javascript"></script>
 <script src="js/footable-0.1.js" type="text/javascript"></script>
</head>

We're not quite finished hooking up the engine. The last thing we have to do is include jQuery, which is itself a javascript library. It's a pretty big one, and was written to make javascript more user friendly. Many libraries -- including FooTable -- are written to work with jQuery, not just standalone javascript.
Instead of downloading jQuery and storing it locally, in this case we're going to link to it. It's such a popular library that Google hosts it here. Have a look at it by following this link, but if you're at all scared of math, be warned. It might give you nightmares. The good thing is you don't need to know exactly how it works to use it. Just like, you know, a car.
<!DOCTYPE HTML>

<head>
<title>Redskins Table Data</title>
<link rel="stylesheet" type="text/css" href="css/style.css" />
<link href="css/footable-0.1.css" rel="stylesheet" type="text/css" />
<link href="css/footable.sortable-0.1.css" rel="stylesheet" type="text/css" />
<script src="http://ajax.googleapis.com/ajax/libs/jquery/1.8.2/jquery.min.js" type="text/javascript"></script>
 <script src="js/footable.sortable.js" type="text/javascript"></script>
<script src="js/footable-0.1.js" type="text/javascript"></script>
</head>

Okay, that's it, engine dropped in and bolted to the frame. Look at your page. It still looks the same! That's because even though the engine is all bolted in, we still haven't hooked up the tubes and hoses and such (I'm not a car guy). We'll do that in the next step.
Step 4: Making the table responsive and sortable
Take a look at the example we're building. It has a lot of information in it. It looks great on a big desktop, but on a mobile phone there are just too many columns.
The table has several purposes.
  • Allow people to explore the schools, see what states had a lot.
  • Allow people to see the the mascots, some of which are kind of crazy looking and more than a little racist.
  • Allow people to see the percentage of Native Americans in each school and the percentage of students from other races.
Our goal is to preserve as much as we can from the table on mobile and tablets, but still be usable. So we want to:
  • Keep the state, the school and the percentage Native American on mobile, but hide the images, which take too long to load.
  • Keep the state, the school, the images and the percentage Native American on tablets
  • Keep the desktop the way it is.
We could do that with CSS, but it would take a long time to write a bunch of media queries. FooTables does the work for us. It has built in three "breakpoints": 480px for a phone, 1024px for a tablet and anything other that is considered a desktop. To hide them, we use data-hide in the table rows. We can do this, because the FooTables code makes this possible. Do this:
<table class="footable">
<thead>
<tr>
<th>State</th>
<th data-hide="phone,tablet">City</th>
<th data-hide="phone">Logo</th>
<th>School</th>
<th>% Native American</th>
<th data-hide="phone,tablet">% White</th>
<th data-hide="phone,tablet">% Hispanic</th>
<th data-hide="phone,tablet">% Black</th>
<th data-hide="phone,tablet">% Asian</th>
<th data-hide="phone,tablet">% 2+ Races</th>
<th data-hide="phone,tablet">No. of Students</th>
</tr>
</thead>
<tbody>
What should happen? Save it and refresh and check it out.
It should look nice, but nothing got hidden? What? We bolted in the engine, we hooked up the hoses and still it doesn't work?
That's because we haven't turned the car on and given it some gas. To do that, we need to write a javascript function -- or a set of instructions -- to tell it to work.
So, add this code in to your head tag
<!DOCTYPE HTML>

<head>
<title>Redskins Table Data</title>
<link rel="stylesheet" type="text/css" href="css/style.css" />
<link href="css/footable-0.1.css" rel="stylesheet" type="text/css" />
<link href="css/footable.sortable-0.1.css" rel="stylesheet" type="text/css" />
<script src="http://ajax.googleapis.com/ajax/libs/jquery/1.8.2/jquery.min.js" type="text/javascript"></script>
<script src="js/footable.sortable.js" type="text/javascript"></script>
<script src="js/footable-0.1.js" type="text/javascript"></script>

<script type="text/javascript">
$(function() {
$('table').footable();
});
</script>

 </head>

That's a very simple function. Don't worry about the semi colons and dollar signs and so forth. Just, for now, know that it says find an element called table (see that anywhere on the page?) and make it a FooTable. That is to say, allow us to do the things to it that FooTable specifically spells out.
If it instead said:
<!DOCTYPE HTML>

<head>
<title>Redskins Table Data</title>
<link rel="stylesheet" type="text/css" href="css/style.css" />
<link href="css/footable-0.1.css" rel="stylesheet" type="text/css" />
<link href="css/footable.sortable-0.1.css" rel="stylesheet" type="text/css" />
<script src="http://ajax.googleapis.com/ajax/libs/jquery/1.8.2/jquery.min.js" type="text/javascript"></script>
<script src="js/footable.sortable.js" type="text/javascript"></script>
<script src="js/footable-0.1.js" type="text/javascript"></script>

<script type="text/javascript">
$(function() {
$('#SeanRocks').footable();
});
</script>

 </head>

...it would look for an element on the page, like a DIV, with an ID of SeanRocks and apply the Footabley properties to it.
This is as simple as these functions get. And these functions are typically where you will tailor these javascript libraries to your specific needs. In our car example, the function is where you'd tell the car to go fast or slow. Or turn right or left.
Anytime you're using a javascript library, you should expect to do something similar with functions.
Okay, give it a try and see if it works. You should be able to sort the table.
